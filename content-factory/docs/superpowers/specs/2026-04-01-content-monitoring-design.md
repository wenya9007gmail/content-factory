# 内容监控工具前端原型设计

## 1. 背景与目标
- 面向内容运营/选题策划团队，帮助他们集中管理多分类的监控任务（关键词/博主/平台）。
- 提供“内容洞察 → 选题分析 → 配置管理”完整链路，支持桌面与移动端。
- 系统默认每日定时运行一次，可手动触发。

## 2. 全局信息架构
1. **顶栏**：品牌、运行状态提示（最近一次任务时间+是否成功）、“立即运行”按钮、AI/系统配置入口。
2. **左侧分类 rail**：
   - 列表包含搜索、分组、徽标（未读洞察、失败警报）。
   - 支持拖拽排序及快捷菜单（重命名/复制/停用/删除）。
   - 新建分类按钮位于列表顶部。
3. **主内容区**：Tab 切换（内容 / 选题分析与报告 / 监控设置）。默认进入“内容”。移动端将 Tab 折叠为顶部胶囊或下拉。
4. **Info Bar**：展示当前分类名称、运行摘要（采集条数、覆盖平台数、错误数）、手动运行按钮。

## 3. 用户关键路径
1. 在左侧切换或创建分类 → Tab1 查看最新内容 → 标记重点条目。
2. 同步跳转 Tab2 → 查看 AI 报告 → 将选题推送到执行计划。
3. Tab3 管理监控项 → 批量启用/禁用平台 → 保存配置。

## 4. Tab 设计
### 4.1 Tab1 内容
- **概览区**（首屏）：
  - 今日亮点卡：采集量、覆盖平台、AI 推荐速览，附“查看报告”跳转 Tab2。
  - 平台热度雷达/堆叠柱：展示各平台爆款占比。点击扇区即等同于筛选平台标签。
- **平台筛选**：
  - 平铺标签（支持多选 + “全部”），标签内显示当日新增量。桌面两行 wrap，移动横向滚动条。
- **时间选择**：
  - 横向日期卡片轮播（最近14天），卡片含日期、热度色块、采集数。默认选最近一次成功运行日。
  - 下方“热力滑尺”可快速拖拽跳转更早日期，标注有报告的日期。
- **时间线视图**：
  - 左侧竖直时间轴，右侧为分日折叠卡组。每个日期 group 展开后显示内容列表。
  - **内容卡模板**：封面、标题、来源平台徽章、关联监控项（#关键词/@博主）、核心指标（点赞/评论/分享/传播指数）、发布时间。
  - 行为按钮：收藏★、标记🔥、加入选题分析、查看原文链接。
  - 支持排序切换（热度、互动率、发布时间）。批量模式允许多选卡片后执行收藏/导出。
- **搜索 & 过滤**：全局搜索框，结合筛选（内容类型、互动区间）。

### 4.2 Tab2 选题分析与报告
- **默认视图**：顶部展示最新报告卡（日期、AI Quick Summary、3条洞察 + “重新分析”按钮）。
- **时间线模式**：
  - 左侧垂直日期列表（7/14/30 天切换）。每个日期卡含总热度、2条摘要、状态指示（成功/失败/未运行）。
  - 右侧主面板内容：
    - AI 选题建议列表（accordion）——字段：方向标题、选题简介（为什么做+爆点+增长空间）、潜力评分、关联内容数量、快速操作按钮（生成提纲、加入执行计划）。
    - 热点回顾：折线图 + top10 列表，展示关注峰值时间段。
    - 重点选题集：将 Tab1 中标记的内容聚合展示。
  - 空状态：某天未运行时展示提示与“立即生成”按钮。
- **选题维度筛选**：
  - 顶部维度条：多选标签（热点/破圈/长尾/竞品/自定义）+ 可输入关键词的搜索框（生成 chip，实时过滤）。
  - 支持保存筛选组合为“视图”，在标签条末端切换。
- **AI 交互**：
  - 每条选题卡右上角显示来源内容数量，点击可预览对应内容卡。
  - “生成推送稿”按钮触发 ChatGPT 衍生指令。

### 4.3 Tab3 监控设置
- **快速模板**：顶部卡片给出常见组合（例如“抖音热门关键词”“小红书品牌对标”），一键创建多条监控项。
- **矩阵配置表**：
  - 行 = 监控项（关键词/博主），列 = 平台列表 + “状态”+“抓取频率”+“AI 深度”。
  - 单元格为状态 pill（启用/暂停），支持批量选区操作；行首图标标识类型（#/@）。
  - 表格顶部提供批量动作浮层：启用、禁用、调整频率、设为重点。
  - 每行显示最近抓取时间、异常提示（如连续失败）。
- **抽屉编辑**：
  - 右侧抽屉包含：基本信息、平台选择、关键词/账号别名、监控频率、AI 分析设置、通知（异常提醒/日报推送）。
  - 底部按钮：“保存”“保存并复制到其他分类”。
- **新增流程**：按钮 → 模态选择类型（关键词/博主/混合）→ 支持批量粘贴，每行生成一个监控项。

## 5. 假数据策略
- 分类示例：Clauecode 选题监控、Vibecoding 选题监控、直播带货矩阵。
- 平台：抖音、小红书、微博、B 站、快手。
- 内容卡样本：
  - 标题《AI 原生剪辑模板 24 小时破 10W 播》、点赞 12.4K、互动率 8.2%、来源抖音、关键词 #AI 剪辑。
  - 标题《Vibecoding 发布开源 Prompt 套件》、点赞 3.1K、来源微博、关联博主 @Vibecoding。
- 选题建议示例：
  - 方向标题《AI 视频模板市场联合评测》；简介：聚焦高热度模板的差异，爆点=“24 小时破 10W 播”，增长空间=品牌联动。
  - 方向标题《对标 claude code 的创作者增长路径》；简介说明用户痛点与机会。
- 监控项示例：
  - #AI剪辑、#选题灵感、#直播脚本。
  - @claudecode_official、@VibecodingLab。

## 6. 技术实现概览（Next.js）
- 框架：Next.js (app router) + Tailwind/Chakra（任选）+ Zustand/Recoil 管理前端状态。优先保持轻量，主要为静态原型。
- 组件层次：
  - `LayoutShell`（顶栏+分类 rail）。
  - `CategoryTabs`（Tab 切换状态保存在 URL search params）。
  - Tab 专属组件（`ContentOverview`, `Timeline`, `ReportPanel`, `MonitoringMatrix` 等）。
  - 假数据通过 `/data/mock.ts` 注入，页面使用 `useEffect` 模拟 API。
- 响应式：基于 CSS grid + `@media` 将左侧 rail 折叠为下拉抽屉，日期卡片变为水平滚动列表。
- 交互细节：
  - 平台筛选使用 `SegmentedControl` 风格按钮，支持多选。移动端使用 `ChipGroup`。
  - 时间轴使用自定义 slider（结合 d3 scale 或简单 canvas）+ 卡片轮播。
  - 矩阵表格可使用 CSS sticky header + 虚拟滚动保证性能。

## 7. AI 与运行提示
- 顶栏 info bar & Tab2 顶部显示运行状态/AI 任务状态。
- “重新分析”按钮触发模拟 API（请求状态+结果 toast）。
- 选题卡上的“生成提纲/推送稿”按钮展示弹窗或右侧浮层，模拟 ChatGPT 输出。

## 8. 开放问题 / 后续迭代
1. 分类数量较多时，是否需要分组/折叠？（当前支持搜索+分组标签，可二期考虑树形结构。）
2. 监控项若超过 1000 条，需要服务端分页和虚拟滚动；原型阶段默认 30-50 条示例。
3. AI 报告是否需要版本管理/对比？（暂留“加载历史”入口，后续再细化。）
